## 💡 密码管理建议

### 方案A：统一初始密码（最简单）

**所有学员初始密码：123456**

**优点：**
- 管理简单
- 易于告知学员

**缺点：**
- 学员可能忘记修改
- 仍有被分享的风险

**适合：**
- 测试阶段
- 学员数量少（<50人）
- 短期使用

---

### 方案B：随机密码（推荐）

**给每个学员生成不同的密码**

示例：
```
学号：20240001 → 密码：Ax7k2M
学号：20240002 → 密码：Bk9p3N
学号：20240003 → 密码：Cq5m8R
```

**优点：**
- 更安全
- 即使泄露一个，其他学员不受影响

**缺点：**
- 管理稍微复杂
- 需要单独告知每个学员

**如何实现：**
在manage.html添加学员时，密码框填入随机密码

---

### 方案C：手机号后6位（折中）

**用学员手机号后6位做密码**

示例：
```
手机号：13812345678 → 密码：345678
手机号：13987654321 → 密码：654321
```

**优点：**
- 学员容易记住
- 相对安全
- 不易分享（别人不知道手机号）

**缺点：**
- 需要收集手机号
- 手机号可能泄露

## 🚨 其他安全风险和解决方案

### 风险1：学员把账号密码都分享了

**现状：**
```
学员A告诉朋友B：
学号：20240001
密码：123456
↓
朋友B也能登录
```

**解决方案：**

#### 短期（1周内可实现）：
1. **使用随机密码** - 分享成本高
2. **定期更换密码** - 比如每月1号统一更换
3. **告知学员违规后果** - 发现后封号

#### 中期（1个月内）：
1. **手机验证码登录** 
   - 每次登录都要短信验证
   - 无法转让
   
2. **设备绑定**
   - 首次登录记录设备
   - 新设备登录需要验证

#### 长期（3个月内）：
1. **使用次数限制**
   - 每个学号每天限制10次对话
   - 异常使用自动提醒
   
2. **IP限制**
   - 检测多个IP使用同一账号
   - 异常登录发送警告

## 📊 监控和检测

### 如何发现盗用？

#### 方法1：使用量异常

**正常使用：**
```
学员A：每天3-5次对话
学员B：每天2-4次对话
```

**异常使用：**
```
学员C：每天50次对话（可能被多人共用）
```

#### 方法2：登录IP分析

**正常：**
```
学员A：始终从重庆IP登录
```

**异常：**
```
学员B：
上午从重庆登录
下午从北京登录
晚上从上海登录
→ 明显被分享了
```
